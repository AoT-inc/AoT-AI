## 1. GIS 기능

### 1.1 개요

AoT의 GIS(Geographic Information System) 기능은 농업 현장에 설치된 센서와 장치의 **위치를 지도에 시각화**하고, 인터랙티브하게 관리할 수 있도록 도와줍니다. 한국 농촌 현장에 최적화된 지도 서비스(VWorld, OpenStreetMap)를 지원합니다.

### 1.2 접근 방법

- **메뉴 경로**: 설정(⚙) → 지도(Layers) 또는 바로 가기 위젯
- **주소**: `/geo/layer` (GIS 레이어 관리)
- **지도 편집기**: `/geo/design` (사이트, 구역, 장치 배치)

### 1.3 주요 기능

#### 지도 기반 센서 위치 시각화
- 각 센서/장치(입력)에 위도/경도 좌표를 등록하면 **지도 위에 마커**로 표시됩니다.
- 마커 색상으로 센서 상태(정상/경고/오류)를 구분할 수 있습니다.
- **클러스터링**: 센서가 밀집된 지역에서는 마커가 자동으로 그룹화됩니다.

#### 인터랙티브 지도 활용
- **확대/축소**: 마우스 휠 또는 버튼으로 확대/축소
- **위치 검색**: 주소 또는 좌표로 현장 위치 검색
- **팝업 창**: 센서 마커를 클릭하면 **최근 측정값, 장치명, 활성화 상태** 확인 가능
- **레이어 토글**: 토양도, 기상 레이어 등 부가 정보 표시/숨기기

#### 다중 센서 관리
- 여러 센서를 **하나의 지도에 동시에** 표시
- **Spatial Join**: 특정 구역(Site/Zone) 안에 속한 센서를 자동으로 감지하고 그룹화
- **파이프 생성**: 관로(배관) 연결을 지도 위에 선으로 시각화
- **레이어 관리**: WMS/TMS 서비스(예: 한국국토정보원 VWorld)를 지도 배경으로 추가

### 1.4 사용 시나리오

> **실생활 예시**: 경기도 오산시 딸기 재배 농장에 10개의 온습도 센서를 설치했을 때,
> 1. 각 센서의 GPS 좌표를 입력하면 지도 위에 빨간색(정상), 노란색(경고) 마커로 표시됩니다.
> 2. 특정 구역의 마커를 클릭하면 최근 24시간 온도 변화 그래프가 팝업에 표시됩니다.
> 3. GIS 레이어에서 VWorld 위성 이미지를 배경으로 설정하면 실제 농장 모습을 확인할 수 있습니다.

### 1.5 외부 API 연동 (프록시)

AoT는 클라이언트 CORS 문제를 해결하기 위해 서버 사이드 프록시를 제공합니다:

| 서비스 | 용도 | 캐시 시간 |
|--------|------|----------|
| RainViewer | 기상 레이더 | 5분 |
| ISRIC SoilGrids | 토양 속성 (pH, 수분 등) | 5분 |
| OpenWeatherMap | 현재 기상 정보 | 5분 |
| Open-Meteo | NASA 위성 데이터 | 5분 |

## 2. Note (메모/알림)

### 2.1 개요

AoT의 **노트(Note)** 시스템은 센서 데이터 외에 **사용자의 관찰 기록, 경고, 작업 메모**를 기록하고 관리하는 기능입니다. GPS 좌표, 파일 첨부, 태그 기능을 통해 농촌 현장의 실질적인 정보를 체계적으로 보관합니다.

### 2.2 접근 방법

- **노트 위젯**: 대시보드의 노트 위젯 (홈 화면에서 바로 접근)
- **장치 노트**: 특정 센서/장치 페이지에서 "메모" 탭 클릭
- **주소**: `/notes` 또는 AJAX API 호출

### 2.3 주요 기능

#### 환경 데이터 기록
- 관찰 내용을 자유 형식 텍스트로 기록 (예: "오늘 오전 10시에 양액 공급 실행, EC 2.0 확인")
- **자동 제목 추출**: 메모의 첫 번째 줄이 자동으로 제목으로 사용됩니다.
- **파일 첨부**: 이미지(PNG, JPG), 동영상(MP4), 문서(PDF, XLSX)를 메모에 첨부 가능
- **GPS 좌표 기록**: 현장의 정확한 위치를 자동으로 저장

#### 태그 시스템
- 메모에 **태그(라벨)**를 부여하여 분류
- 예약 태그: `widget`(위젯 표시), `map_hidden`(지도 숨김), `alert`(경고)
- 예: 딸기 재배 → `#작물`, `#관찰`, `#양액`

#### Alert/경고 시스템
- 특정 조건이 충족되면 **사용자에게 경고를 표시**합니다.
- **우선순위**: 일반(0), 높음(1), 심각(2)
- 경고 시 노트에 `alert` 태그가 자동 부여
- **이메일/알림**: SMTP 설정 시 경고 내용을 이메일로 발송 가능 (설정 → SMTP에서 구성)

#### 사용자 알림 설정
- **설정 → 알림**: 시스템 전체 알림 수신 여부, 알림 소리, 이메일 수신 설정
- **장치별 알림**: 특정 센서/장치에 대한 알림만 활성화/비활성화
- **시간대별**: 야간(밤 10시~오전 6시) 알림 자동 묵음 설정 가능

### 2.4 REST API

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/notes/target/<target_id>` | 특정 장치의 모든 노트 조회 |
| GET | `/notes/geo` | GPS 좌표가 있는 모든 노트 조회 (지도 표시용) |
| POST | `/notes/create` | 새 노트 생성 |
| POST | `/notes/update/<unique_id>` | 노트 수정 |
| DELETE | `/notes/delete/<unique_id>` | 노트 삭제 |
| POST | `/notes/toggle_map_visibility` | 지도 표시/숨김 전환 |
| GET | `/notes/tags` | 모든 태그 목록 조회 |

### 2.5 사용 시나리오

> **실생활 예시**: 봄철 고추 재배 중 gray mold 발견 시,
> 1. 스마트폰으로 병든 잎을 촬영하여 노트에 사진 첨부
> 2. "고추 gray mold 의심, 상대습도 85% 이상 지속" 메모 작성 → GPS 자동 기록
> 3. 태그: `#작물=#고추`, `#병해=#gray_mold`, `#관찰`
> 4. 이후 같은 위치에서 비슷한 관찰이 반복되면 AI가 자동으로 패턴을 감지하여 경고 발송

## 3. Input / Output / Function

### 3.1 개요

AoT의 핵심은 **센서(입력) → 데이터 처리 → 장치(출력) 제어** 데이터 플로우입니다. Function(함수)은 이 사이에 로직을 끼워 넣어 자동화된 농업 관리를 가능하게 합니다.

### 3.2 입력(Input) — 센서 데이터 수집

#### 접근 방법
- **메뉴**: 설정(⚙) → 입력(Input)
- **주소**: `/input`

#### 센서 추가 절차
1. **입력 추가** 버튼 클릭
2. **센서 유형** 선택 (예: DHT22, DS18B20, EC-5 등 100개 이상)
3. **연결 방식** 설정: GPIO 핀, I2C 주소, UART 등
4. **측정 주기** 설정 (예: 60초마다)
5. **활성화** 버튼 클릭

#### 주요 센서 유형

| 센서 유형 | 측정 항목 | 농업 활용 |
|---------|---------|---------|
| DHT22 | 온도, 습도 | 하우스 환경 모니터링 |
| DS18B20 | 온도 | 수경재배 수온 관리 |
| EC-5 (Soil Moisture) | 토양 수분 | 관개 자동화 |
| pH EZO | pH | 양액 관리 |
| Atlas EC | 전기전도도(EC) | 영양액 모니터링 |
| BH1750 | 조도 | 보광 등 환경 제어 |
| ADS1115 | 아날로그 전압 | 아날로그 센서 확장 |

### 3.3 출력(Output) — 장치 제어

#### 접근 방법
- **메뉴**: 설정(⚙) → 출력(Output)
- **주소**: `/output`

#### 제어 방식

| 출력 유형 | 설명 | 농업 활용 |
|---------|------|---------|
| GPIO On/Off | 릴레이로 장치 전원 제어 | 관개 밸브, 히터, 팬 |
| PWM | 세기 조절 (0~100%) | LED 보광, DC 팬 |
| 연동 펌프 | 액체 분배 | 양액 자동 공급 |
| 315/433 MHz | 무선 원격 제어 | 원격 관개 |
| Linux/Python 명령 | 시스템 명령 실행 | 자동화 스크립트 |

#### PWM 활용 예시 (DC 팬 제어)
- **주파수**: 25kHz (부드러운 속도 조절)
- **듀티 사이클**: 40%~80% (여름철 온도별 속도 조절)

> **실생활 예시**: 여름철 온도에 따라 자동으로 팬 속도가 조절됩니다.
> - 28°C 이하: 40% 속도
> - 30°C~33°C: 60% 속도
> - 33°C 이상: 80% 속도

### 3.4 함수(Function) — 자동화 로직

#### 접근 방법
- **메뉴**: 설정(⚙) → 함수(Function)
- **주소**: `/function`

#### 주요 함수 유형

| 함수 유형 | 설명 | 예시 |
|---------|------|------|
| **PID 제어** | 온도/습도 등을 목표값으로 자동 조절 | 온도 25°C 유지 → 히터/쿨러 자동 제어 |
| **조건부(Conditional)** | 조건 충족 시 특정 출력 동작 | "습도 < 50%이면 관개 펌프 On" |
| **트리거(Timer)** | 정해진 시간에 동작 | "매일 06:00에 관개 시작" |
| **스케줄러(Scheduler)** | 복잡한 일정 관리 | "비 오는 날에는 관개 건너뛰기" |

### 3.5 데이터 플로우 전체 구조

```
[센서] ──(측정 주기)──▶ [입력 컨트롤러] ──▶ [InfluxDB 시계열 DB]
  (Input)                              │
   │  GPIO/I2C/UART              측정값 저장
   ▼                               │
 [출력 컨트롤러] ◀── [함수/PID/Conditional] ◀── [사용자 설정/AI]
  (Output)                                │
   │  GPIO On/Off, PWM, 펌프        자동화 로직
   ▼                                │
 [실제 장치] ◀── 장치 제어 명령         │
 (릴레이, 밸브, 히터 등)                │
                                    ▼
                        [AI 에이전트/스케줄러]
                          (자동 의사결정)
```

### 3.6 주요 API 엔드포인트

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/geo/devices` | 모든 장치 목록 조회 |
| GET | `/api/inputs` | 입력(센서) 목록 |
| POST | `/input_submit` | 입력 추가/수정/삭제 |
| GET | `/api/outputs` | 출력(제어) 목록 |
| POST | `/output_submit` | 출력 추가/수정/삭제 |
| GET | `/output_started_at_public/<device_id>/<ch>` | 장치 작동 시간 조회 |

## 4. AI 기능

### 4.1 개요

AoT는 **AI 에이전트(자연어 명령 처리), 이상 탐지, 자동 스케줄링** 등 AI 기반 지능형 농업 관리를 지원합니다. Gemini, GPT-4o 등 다양한 AI 모델과 연동할 수 있습니다.

### 4.2 접근 방법

- **메뉴**: 설정(⚙) → AI (인공지능)
- **주소**: `/ai` (AI 대시보드)
- **에이전트 관리**: `/ai/agent`
- **모니터링**: `/ai/manage`

### 4.3 AI 에이전트 시스템

#### 에이전트란?
AI 에이전트는 **자연어로 명령을 내려 자동으로 시스템이 작동하게 하는 역할**입니다.

#### 에이전트 추가 방법
1. **AI → 에이전트** 페이지에서 "새 에이전트 추가" 클릭
2. **AI 서비스 선택**: Gemini, GPT-4o, Claude 등
3. **API 키 입력**: 해당 AI 서비스의 API 키 설정
4. **역할(Role) 설정**: router, planner, worker, synthesizer, executor

#### 자연어 명령 예시
> **사용자 입력**: "오늘 오후 3시부터 6시 사이 온도가 30도를 넘으면 자동으로 팬을 80%로 작동시켜줘"
>
> **AI 에이전트 처리**:
> 1. router가 명령을 해석
> 2. planner가 조건부 함수 설정 내용 작성
> 3. worker가 실행 → 조건부 함수 자동 생성
> 4. synthesizer가 결과 보고서 작성

### 4.4 이상 탐지 및 예측

#### 이상 탐지 (Anomaly Detection)
- 센서 데이터의 **평소 패턴에서 벗어나는 이상값**을 자동 감지
- 예: "평소 토양 수분이 40% 수준인데 갑자기 5%로 급감 → 센서 오류 또는 물 공급 문제 감지"
- 감지 시: **자동 경고 발송 + 노트 기록**

#### 예측 시스템
- 과거 데이터를 기반으로 **미래 환경 변화를 예측**
- 예: "최근 3일간 온도가 하루에 1.5°C씩 상승 중 → 5일 후에는 35°C 도달 예상"
- 예측 결과: **미리 관개량/환기량 자동 조절**

### 4.5 자동 스케줄링

#### AI 스케줄러
- AI가 재배 작물에 맞는 **최적의 관개/환기 스케줄**을 자동 생성
- 외부 데이터(기상 예보) 연동 가능

#### Gantt 차트 (AI 스케줄러)
- AI가 생성한 작업 일정을 **가로 눈금 차트(Gantt)**로 시각화
- `/ai/manage` 페이지의 "manage" 탭에서 확인 가능

### 4.6 자기 학습 시스템

#### 경험 지식 그래프 (EKG)
- AI가 **과거 작업 결과를 자동으로 학습**하여 다음 결정 품질을 높임
- 학습 데이터: 사용자가 승인/거부한 AI 제안, 실행 결과, 환경-수확량 상관관계
- **피드백 루프**: 사용자가 AI 결과를 평가 → AI가 이를 반영하여 개선

## 5. 사용자 편의성

### 5.1 웹 대시보드 사용법

#### 대시보드 개요
- AoT의 **메인 화면**으로 로그인 후 바로 접속
- **주소**: `/dashboard` 또는 메인 URL (`/`)
- 위젯(카드) 형태로 다양한 정보를 한눈에 확인

#### 주요 위젯

| 위젯 | 내용 |
|------|------|
| **AoT Map** | GIS 지도 + 센서 현황 |
| **Notes** | 최근 메모/경고 |
| **Data** | 센서 측정값 차트 |
| **Energy** | 에너지 사용량 |
| **PID** | PID 제어 상태 |
| **Camera** | 카메라 영상 (연결된 경우) |
| **I/O** | 입력/출력 상태 요약 |

#### 대시보드 커스터마이징
- **그리드 레이아웃**: 위젯 위치 Drag & Drop으로 자유롭게 배치
- **위젯 추가/삭제**: `[+] 추가` / `[×]` 삭제
- **다중 탭**: 목적별로 대시보드를 여러 개 생성 가능

### 5.2 다국어 지원

#### 언어 변경 방법
- **로그인 화면**: 언어 선택 드롭다운
- **설정 → 일반 → 언어**: 로그인 후 변경 가능

### 5.3 반응형 UI (모바일 접근성)

- **모바일 최적화**: 스마트폰/태블릿에서도 AoT UI를 정상적으로 사용 가능
- **모바일 메뉴**: 햄버거 메뉴(≡)로Navigation Drawer 지원
- **모바일에서 확인 가능한 정보**: 센서 현황, 출력 상태, 최근 알림, 카메라 영상

### 5.4 Docker 기반 쉬운 배포

### 6.1 현재 작동하지 않는 기능

- **상태**: RainViewer 공개 API v2가 **상업 중단**됨 (2026년 현재)
- **대안**: OpenWeatherMap 또는 Open-Meteo의 강수 예보 데이터 사용
- **우선순위**: 중간

- **상태**: `/ai/scheduler` → `/scheduler`로 **301 리다이렉트**
- **대안**: `/ai/manage?tab=manage`에서 AI 작업 히스토리 확인
- **우선순위**: 낮음

#### AI 에러 대시보드
- **상태**: `/ai/errors` → `/ai/manage?tab=errors`로 **301 리다이렉트**
- **우선순위**: 낮음

### 6.2 개선이 필요한 부분
