## 1. 배경

GIS 지도 위에 시설 도형(폴리곤)을 그려 표현하는 인프라는 이미 존재한다(`AoT_map` 위젯 + `GeoShape type='facility'`). 그러나 현재는 단순 평면 폴리곤일 뿐이며, 시설의 **3D 형상·외피 구성·창호·액추에이터·작물**과 같은 운영 메타데이터를 보유하지 않는다.

본 작업은 (a) GEO 페이지에서 시설을 **표준화된 사양으로 등록**하게 하고, (b) 그 사양으로부터 **기계설비 1차 용량 산정 근사치**를 자동 도출하며, (c) 대시보드 위젯에서 **AI 권고와 함께 운영 미믹**을 제공하기 위한 데이터·UI 기반을 마련한다.

## 2. 목적

| # | 목적 | 측정 방법 |
|---|---|---|
| G1 | 시설을 GIS 위에 도형으로 등록하면서 형상·외피·창호·액추에이터 사양을 함께 입력 | `/geo/facility` 페이지에서 등록 → DB에 1건 저장 |
| G2 | 등록된 사양으로부터 바닥면적·체적·창호면적·환기량·난방/냉방 1차 부하 자동 산출 | 등록 직후 미리보기 노출 |
| G3 | 대시보드 위젯에서 시설 단면(미믹)·환경값·AI 권고를 한 화면에 제공 | `AoT_facility` 위젯 클릭→3섹션 렌더 |
| G4 | AI는 권고만 제시하고 사용자의 승인 클릭으로 액추에이터를 실행 | "승인하고 적용" 버튼 클릭 시 output 명령 발행 |

## 3. 사용자 시나리오

### 3-1. 시설 등록 시나리오 (GEO 페이지)
1. 사용자는 `/geo/design` 페이지에서 *"Facility Design"* 버튼을 클릭한다.
2. `/geo/facility` 페이지로 이동한다.
3. 베이스 지도에서 외곽을 그려 단동/연동 여부와 동 수를 선택한다.
4. 시설 타입(표준 온실 등 프리셋)과 치수, 외피(층수·자재), 창호, 커튼, 액추에이터를 매핑한다.
5. 우측 미리보기에서 자동 산출된 면적·체적·환기량·부하를 확인한다.
6. 저장 → 지도(AoT_map)와 위젯(AoT_facility)에서 즉시 사용 가능.

### 3-2. 운영 시나리오 (대시보드 위젯)
1. 운영자는 대시보드의 `AoT_facility` 위젯에서 시설을 선택한다.
2. SVG 단면 미믹에서 측창·천창·커튼·관수의 현재 상태(실선)와 AI 권고 상태(점선)를 비교한다.
3. AI 권고 카드(즉시·1h·6h)를 읽고, 근거·예상효과를 확인한다.
4. *"승인하고 적용"* 클릭 시 매핑된 output에 명령이 발행된다.
5. 1시간 후 카드는 예상치 vs 실측을 자동으로 비교 표시한다.

## 4. 기능 요구사항

### 4-1. 시설 등록(GEO 페이지)

- **F-REG-1**: `/geo/design` 페이지의 `GIS Settings` 버튼 **왼쪽**에 `Facility Design` 버튼을 추가한다.
- **F-REG-2**: 클릭 시 `/geo/facility` 페이지로 **이동**한다(모달 X, iframe X).
- **F-REG-3**: 시설 등록 워크플로는 5단계로 구성된다: 지도 선택 → 외곽 도형 → 사양 입력 → 미리보기 → 저장.
- **F-REG-4**: 단동(`structure=single`)은 외곽 폴리곤 1개로 등록된다.
- **F-REG-5**: 연동(`structure=connected`)은 외곽 폴리곤 1개 + 동 수 입력(N≥2) → 내부 분동선이 자동 생성되거나 수동 그릴 수 있다.
- **F-REG-6**: 시설 프리셋 *"표준 온실(둥근 아치형 비닐하우스)"* 의 기본 치수는 가로폭 7m, 측고 2m, 용마루 4m이며, 사용자가 변경 가능하다.

### 4-2. 외피·창호·커튼 사양

- **F-ENV-1**: 외피 층수(`layer_count`)는 1 또는 2를 선택할 수 있다.
- **F-ENV-2**: 외피 자재 후보는 `vinyl_single`, `vinyl_double`, `po_film`, `polycarbonate`, `glass` 5종이다.
- **F-ENV-3**: 내피 자재 후보는 `vinyl_single`, `non_woven_fabric`, `pe_film`, `polycarbonate`, `air_cushion` 5종이다.
- **F-ENV-4**: `layer_count=2`인 경우 내피의 측창·천창 제어 모드를 `synced` 또는 `independent`로 선택할 수 있다.
- **F-ENV-5**: 커튼은 보온커튼(`thermal`)과 차광막(`shade`) 2종이며 외피·내피와 별개 레이어로 취급한다.
- **F-ENV-6**: 측창·천창은 외피·내피 각각 활성/비활성을 선택할 수 있다.

### 4-3. 액추에이터 매핑

- **F-ACT-1**: 시설은 다음 12종의 액추에이터 슬롯을 가진다(매핑 안 하면 `null`):
  - 환기: `outer_side_vent_motor`, `outer_roof_vent_motor`, `inner_side_vent_motor`, `inner_roof_vent_motor`
  - 커튼: `thermal_curtain`, `shade_curtain`
  - 관수: `irrigation_valve`
  - 팬: `circulation_fan`, `exhaust_fan`
  - 냉/난방: `heater`, `cooler`, `heat_pump`
- **F-ACT-2**: 각 슬롯은 기존 Mycodo `output` 또는 `function`의 `unique_id`를 참조한다.
- **F-ACT-3**: 매핑된 액추에이터만 미믹·AI 권고에 노출된다.

### 4-4. 용량 자동 산출(참고용)

- **F-CALC-1**: 등록 직후 다음 항목을 자동 계산해 `computed` 필드에 캐시한다:
  - 바닥면적(m²), 외피면적(m²), 체적(m³)
  - 창호 개구면적(m², 측창·천창 합산), 글레이징 면적(m²)
  - 1차 환기량(m³/h, 자연환기 + 강제환기 합산)
  - 1차 난방부하(kW), 1차 냉방부하(kW)
- **F-CALC-2**: 산출 결과는 *"기계설비 1차 산정 참고치"* 라벨과 함께 표시한다(±5~10% 근사치 명시).
- **F-CALC-3**: 자재별 열관류율(U)·일사투과율 테이블은 시스템에 내장(편집 가능)한다.

### 4-5. 대시보드 위젯(`AoT_facility`)

- **F-WIDGET-1**: `AoT_map.py`와 동일한 `WIDGET_INFORMATION` dict 패턴으로 구현한다.
- **F-WIDGET-2**: 위젯 본체는 3섹션으로 구성: A) 시설 미믹 SVG, B) 환경 요약 + 미니 타임라인, C) AI 권고 카드.
- **F-WIDGET-3**: 단동은 단면 1개, 연동은 동별 단면을 가로로 나열한다.
- **F-WIDGET-4**: 이중 외피인 경우 외피·내피 단면을 분리해서 표시하며, 사이 공기층을 음영으로 표시한다.
- **F-WIDGET-5**: 액추에이터 현재 상태는 실선, AI 권고 상태는 점선으로 동시 표시한다.

### 4-6. AI 권고

- **F-AI-1**: AI는 권고만 제시하며 사용자의 *승인 클릭*으로만 액추에이터에 명령이 발행된다(자동 실행 금지).
- **F-AI-2**: 권고 카드는 horizon별 3종을 제공: 즉시(now), 1시간 후(1h), 6시간 후(6h).
- **F-AI-3**: 각 카드는 다음 정보를 반드시 포함: 권고 액션, 근거(외기·내부·작물 조건 1~2줄), 예상효과(수치), 신뢰도.
- **F-AI-4**: 권고 액추에이터 우선순위는 에너지 효율 순서로 적용한다: 자연환기(창호) → 커튼 → 순환팬 → 배기팬 → 냉/난방기.
- **F-AI-5**: 사용자는 *승인*, *수정*, *무시* 중 하나를 선택할 수 있으며, *수정* 시 슬라이더로 미세조정 후 적용한다.
- **F-AI-6**: 적용 1시간 후 *예상치 vs 실측* 차이가 카드에 자동 표시된다(학습 피드백 루프).

## 6. 범위 / PoC

### 6-1. PoC 범위(in-scope)
- 표준 온실(둥근 아치형 비닐하우스) 프리셋 1종
  - 단동: 가로 7m × 측고 2m × 용마루 4m
  - 연동: 동 수 N 입력 → footprint 자동 확장 + 내부 분동선
- 단층/이중 외피 모두 지원
- 액추에이터 12종 매핑(매핑 안 한 슬롯은 비표시)
- AI 권고 horizon 3종(now/1h/6h) — 룰베이스 또는 LLM 무관

### 6-2. 차기(out-of-scope)
- 비닐하우스 단동/연동, 창고 등 다른 시설 타입의 SVG 미믹 템플릿
- 작물별 ETc, 단열 정밀 보정 등 ±3% 정밀 산출
- 자동 액추에이터 실행(무인 운전)
- 3D 외관 렌더링(glTF) — 압출 fill-extrusion만 지원

## 7. 데이터 저장 결정

- **신규 테이블**: `geo_facility`
- **외곽 폴리곤**: 기존 `geo_shape` 테이블에 `type='facility'`로 저장(현 시스템 그대로)
- **연동 동별 분동선**: `geo_shape`에 `type='facility_bay'`, `parent_id`=외곽 facility shape id로 저장
- **연결 키**: `geo_facility.shape_uuid` → `geo_shape.unique_id`

## 8. 의존성·전제

- MapLibre GL JS 3.6.2 (Pure 모드)
- Mycodo 위젯 프레임워크 (`WIDGET_INFORMATION` dict)
- 기존 `routes_geo.py` blueprint, `geo_overlays.py` 시스템
- 기존 Output / Function 모델(액추에이터 매핑 대상)
- Alembic 마이그레이션 시스템

## 9. 합의된 결정사항(누적)

1. 버튼 호출 방식: **페이지 이동**
2. 시설 데이터 저장: **신규 `geo_facility` 테이블**
3. 데이터 모델: **Parent-Child 모델** (단동·연동 통합)
4. 외피 구조: **단층·이중 모두 지원**, 내피 제어는 `synced`/`independent`
5. 자재 후보: 외피 5종 + 내피 5종(`polycarbonate`, `air_cushion` 포함)
6. 액추에이터: 12종(팬·냉난방기 포함)
7. AI 권한: **권고 + 승인 버튼**까지만(자동 실행 금지)
8. PoC 시작: 표준 온실 단동 1개 → 연동 → 타 시설 타입(단계적)

## 10. 미해결 사항

- 표준 온실 외 시설(비닐하우스 단동/연동, 창고)의 PoC 진입 시점
- AI 추론 엔진(룰베이스 vs LLM) 선택 — 본 PRD 범위 외
- 자재별 U·투과율 테이블의 초기값 출처(설계 단계 기본값 적용, 추후 실측 교정)
