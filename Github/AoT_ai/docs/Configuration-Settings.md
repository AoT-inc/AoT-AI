## 시계열 데이터베이스 설정

Page\: `[관리] -> 시스템 관리 -> 일반 설정`

측정값은 시계열 데이터베이스에 저장됩니다. 현재 AoT에서 사용할 수 있는 옵션은 InfluxDB 1.x와 InfluxDB 2.x입니다. InfluxDB 1.x는 32비트 및 64비트 운영 체제 모두에서 작동하지만, 2.x는 64비트 운영 체제에서만 작동합니다. 따라서 32비트 운영 체제를 사용하는 경우 InfluxDB 1.x를 사용해야 합니다. AoT 설치 중 influxDB 1.x, 2.x 또는 설치하지 않음을 선택할 수 있습니다. InfluxDB를 설치하지 않은 경우에는 AoT가 측정값을 저장하고 조회할 수 있도록 별도의 설치 주소와 인증 정보를 지정해야 합니다.

Docker로 설치할 경우, 설치 후 InfluxDB Docker 컨테이너에 연결하기 위해 호스트 이름을 "aot_influxdb"로 변경해야 합니다.

## 에너지 사용 설정

Page\: `[관리] -> 시스템 관리 -> 일반 설정`

정확한 에너지 사용 통계를 계산하려면, 전기 시스템의 몇 가지 특성을 알아야 합니다. 이 변수들은 릴레이를 통해 전기 장치를 작동시키는 전기 시스템의 특성을 설명해야 합니다.

!!! note
    전류 센서를 사용하지 않는 경우, 정확한 에너지 사용 계산을 위해 각 출력에 대해 정확한 전류 사용량을 설정해야 합니다 ([출력 설정](Outputs.md) 참조).

<table>
<thead>
<tr class="header">
<th>설정</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>최대 전류(A)</td>
<td>한 번에 허용되는 최대 전류를 설정합니다. 출력이 켜질 경우 활성 장치들의 총합이 이 값을 초과하면 손상을 방지하기 위해 출력을 켤 수 없습니다.</td>
</tr>
<tr>
<td>전압(V)</td>
<td>출력을 통해 스위칭되는 교류(AC) 전압입니다. 일반적으로 120 또는 240입니다.</td>
</tr>
<tr>
<td>kWh당 비용</td>
<td>kWh당 전기 요금을 입력합니다.</td>
</tr>
<tr>
<td>통화 단위</td>
<td>전기 요금에 사용되는 통화 단위입니다.</td>
</tr>
<tr>
<td>검침일</td>
<td>전기 계량기를 확인하는 날짜(1~30일)입니다. 전기 요금 청구일과 일치해야 합니다.</td>
</tr>
<tr>
<td>사용량/비용 보고서 생성</td>
<td>에너지 사용량 보고서를 생성할 시점을 정의합니다. 현재는 출력 시간 기반 계산 방식만 지원됩니다.</td>
</tr>
<tr>
<td>보고서 생성 주기</td>
<td>사용량/비용 보고서를 자동으로 생성할 주기를 설정합니다.</td>
</tr>
<tr>
<td>보고서 생성 요일/일자</td>
<td>보고서를 생성할 요일(Daily: 1~7, 월요일=1) 또는 월별 생성일(1~28)을 설정합니다.</td>
</tr>
<tr>
<td>보고서 생성 시간(시)</td>
<td>보고서를 생성할 시간을 0~23 사이에서 설정합니다.</td>
</tr>
</tbody>
</table>

## 컨트롤러 샘플링 주기 설정

## 측정값 설정

Page\: `[관리] -> 시스템 관리 -> 측정`

AoT의 내장 타입 및 수식을 넘어 기능을 확장할 수 있도록 새로운 측정값, 단위, 변환식을 생성할 수 있습니다. 측정값을 만들기 전에 단위를 먼저 생성해야 하며, 이는 측정값을 생성할 때 단위를 선택해야 하기 때문입니다. 이미 존재하는 측정값에 추가 단위를 지정할 수도 있습니다. 예: `altitude`는 이미 존재하지만 `fathom` 단위를 추가하고 싶다면 먼저 `fathom` 단위를 만든 후, `fathom`이 선택된 상태로 `altitude` 측정값을 생성하면 됩니다.

<table>
<thead>
<tr class="header">
<th>설정</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>측정값 ID</td>
<td>입력 모듈의 measurements_dict에서 사용할 측정값의 ID (예: &quot;length&quot;, &quot;width&quot;, &quot;speed&quot;)</td>
</tr>
<tr>
<td>측정값 이름</td>
<td>측정값의 일반 이름 (예: &quot;길이&quot;, &quot;무게&quot;, &quot;속도&quot;)</td>
</tr>
<tr>
<td>측정값 단위</td>
<td>측정값과 관련된 단위를 모두 선택합니다.</td>
</tr>
<tr>
<td>단위 ID</td>
<td>입력 모듈의 measurements_dict에서 사용할 단위의 ID (예: &quot;K&quot;, &quot;g&quot;, &quot;m&quot;)</td>
</tr>
<tr>
<td>단위 이름</td>
<td>단위의 일반 이름 (예: &quot;킬로그램&quot;, &quot;미터&quot;)</td>
</tr>
<tr>
<td>단위 약어</td>
<td>단위의 약어 (예: &quot;kg&quot;, &quot;m&quot;)</td>
</tr>
<tr>
<td>변환 기준 단위</td>
<td>변환의 기준이 되는 단위</td>
</tr>
<tr>
<td>변환 대상 단위</td>
<td>변환할 단위</td>
</tr>
<tr>
<td>변환식</td>
<td>한 단위를 다른 단위로 변환하는 데 사용할 식입니다. 소문자 &quot;x&quot;가 포함되어야 하며 (예: &quot;x/1000+20&quot;, &quot;250*(x/3)&quot;), 실제 측정값이 해당 x로 대체됩니다.</td>
</tr>
</tbody>
</table>

## 사용자 설정

Page\: `[관리] -> 시스템 관리 -> 사용자`

AoT의 로그인 시스템이 활성화되기 위해서는 최소한 하나의 관리자(Admin) 사용자가 필요합니다. 관리자 사용자가 존재하지 않으면, 웹 서버는 관리자 생성 폼으로 리디렉션됩니다. 이는 AoT를 처음 시작할 때 표시되는 첫 번째 페이지입니다. 관리자 사용자가 생성된 후에는 사용자 설정 페이지에서 추가 사용자를 생성할 수 있습니다.

<td>2자에서 64자 사이의 사용자 이름을 선택합니다. 대소문자를 구분하지 않으며 모두 소문자로 변환됩니다.</td>

## 라즈베리파이 설정

Page\: `[관리] -> 시스템 관리 -> 라즈베리파이`

라즈베리파이 설정은 AoT가 실행되는 Linux 시스템의 일부를 구성합니다.

PWM 출력 및 PWM, RPM, DHT22, DHT11, HTU21D 입력을 사용하려면 pigpiod가 필요합니다.

<table>
<thead>
<tr class="header">
<th>설정</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>기능 활성화/비활성화</td>
<td><code>raspi-config</code> 명령을 통해 웹 UI에서 활성화 및 비활성화할 수 있는 시스템 인터페이스입니다.</td>
</tr>
<tr>
<td>pigpiod 샘플링 속도</td>
<td>pigpiod 서비스가 작동할 샘플링 속도입니다. 숫자가 낮을수록 더 빠른 PWM 주파수를 사용할 수 있지만, Pi Zero에서는 프로세서 부하가 크게 증가할 수 있습니다. pigpiod는 필요하지 않은 경우 완전히 비활성화할 수도 있습니다 (위의 참고 사항 참조).</td>
</tr>
</tbody>
</table>

## 카메라 설정

Page\: `[관리] -> 시스템 관리 -> 카메라`

AoT는 여러 대의 카메라를 동시에 사용할 수 있습니다. 각 카메라는 카메라 설정에서 설정한 후 소프트웨어 전반에서 사용할 수 있습니다.

!!! note
    제조사별 하드웨어 및 소프트웨어 차이로 인해 일부 옵션(예: 색조 또는 화이트 밸런스)은 특정 카메라에서 사용할 수 없을 수 있습니다.

<thead>
<tr class="header">
<th>설정</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>유형</td>
<td>카메라가 Raspberry Pi 카메라인지 USB 카메라인지 선택합니다.</td>
</tr>
<tr>
<td>라이브러리</td>
<td>카메라와 통신하기 위해 사용할 라이브러리를 선택합니다. Raspberry Pi 카메라는 picamera를 사용하며, USB 카메라는 fswebcam으로 설정해야 합니다.</td>
</tr>
<tr>
<td>장치</td>
<td>카메라에 연결할 장치를 지정합니다. fswebcam 라이브러리만 이 옵션을 사용합니다.</td>

<td>스틸 이미지 캡처(타임랩스 포함) 중에 활성화될 출력을 설정합니다.</td>

<tr>
<td>출력 지속 시간</td>
<td>이미지 캡처 전에 출력이 활성화될 시간을 설정합니다.</td>

## 진단 설정

Page\: `[관리] -> 시스템 관리 -> 진단`

시스템에서 비호환 구성으로 인해 문제가 발생할 수 있습니다. 이는 시스템의 일부(입력, 출력 등)가 잘못 구성되었거나 데이터베이스 업그레이드가 제대로 처리되지 않은 업데이트, 또는 기타 예기치 못한 문제로 인해 발생할 수 있습니다. 문제의 원인을 파악하거나 문제 자체를 해결하기 위해 진단 작업을 수행해야 할 때가 있습니다. 아래 옵션은 문제를 완화하기 위한 것입니다. 예를 들어, `데이터 -> 대시보드` 페이지에서 오류를 일으키는 잘못 구성된 대시보드 요소로 인해 해당 페이지에 접근할 수 없는 경우, 모든 대시보드 요소를 삭제하는 것이 다시 접근할 수 있는 가장 경제적인 방법일 수 있습니다. 단, 이 경우 기존의 모든 대시보드 요소를 다시 추가해야 합니다.

<table>
<thead>
<tr class="header">
<th>설정</th>
<th>설명</th>
</tr>
</thead>
<tbody>
<tr>
<td>모든 대시보드 삭제</td>
<td>데이터 -> 대시보드 페이지에 저장된 모든 대시보드를 삭제합니다.</td>
</tr>
<tr>
<td>모든 입력 삭제</td>
<td>설정 -> 입력 페이지의 모든 입력을 삭제합니다.</td>
</tr>
<tr>
<td>모든 노트 및 태그 삭제</td>
<td>더보기 -> 노트 페이지의 모든 노트와 태그를 삭제합니다.</td>
</tr>
<tr>
<td>모든 출력 삭제</td>
<td>설정 -> 출력 페이지의 모든 출력을 삭제합니다.</td>
</tr>
<tr>
<td>설정 데이터베이스 삭제</td>
<td>aot.db 설정 데이터베이스를 삭제합니다 (경고: 이 작업은 모든 설정과 사용자를 삭제합니다).</td>
</tr>
<tr>
<td>파일 삭제: .dependency</td>
<td>.dependency 파일을 삭제합니다. 종속성 설치 페이지에 접근하는 데 문제가 있는 경우 시도해 보세요.</td>
</tr>
<tr>
<td>파일 삭제: .upgrade</td>
<td>.upgrade 파일을 삭제합니다. 업그레이드 페이지에 접근하거나 업그레이드를 실행하는 데 문제가 있는 경우 시도해 보세요.</td>
</tr>
<tr>
<td>InfluxDB 1.x 데이터베이스 재생성</td>
<td>InfluxDB 1.x 측정 데이터베이스를 삭제한 후 재생성합니다. 이 작업은 모든 측정 데이터를 삭제합니다!</td>
</tr>
<tr>
<td>InfluxDB 2.x 데이터베이스 재생성</td>
<td>InfluxDB 2.x 측정 데이터베이스를 삭제한 후 재생성합니다. 이 작업은 모든 측정 데이터를 삭제합니다!</td>
</tr>
<tr>
<td>이메일 카운터 초기화</td>
<td>시간당 이메일 카운터를 초기화합니다.</td>
</tr>
<tr>
<td>종속성 설치</td>
<td>AoT 시스템 전체에 필요한 모든 종속성을 설치하는 스크립트를 시작합니다.</td>

</tr>
