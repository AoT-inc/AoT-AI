페이지: `설정 -> 입력`

지원되는 입력 장치의 전체 목록은 [지원되는 입력 장치](Supported-Inputs.md)를 참조하세요.

센서, ADC 신호 또는 명령의 응답과 같은 입력은 환경 또는 기타 위치의 조건을 측정하여 시계열 데이터베이스(InfluxDB)에 저장합니다. 이 데이터베이스는 [대시보드 위젯](Data-Viewing.md#dashboard), [함수](Functions.md) 및 AoT의 다른 부분에서 작동하는 데 필요한 측정값을 제공합니다. 입력을 추가, 구성 및 활성화하여 데이터베이스에 측정값을 기록하고 AoT 전반에서 사용할 수 있도록 하세요.

### 사용자 정의 입력

[사용자 정의 입력 모듈 빌드](https://github.com/AoT-inc/AoT-AI/wiki/Building-a-Custom-Input-Module) Wiki 페이지를 참조하세요.

AoT에는 사용자 정의 입력을 생성하여 AoT 시스템에서 사용할 수 있도록 하는 사용자 정의 입력 가져오기 시스템이 있습니다. 사용자 정의 입력은 `[기어 아이콘] -> 구성 -> 사용자 정의 입력` 페이지에서 업로드 및 가져오기할 수 있습니다. 가져온 후에는 `설정 -> 입력` 페이지에서 사용할 수 있습니다.

작동하는 입력 모듈을 개발한 경우, [새 GitHub 이슈 생성](https://github.com/AoT-inc/AoT-AI/issues/new?assignees=&labels=&template=feature-request.md&title=New%20Module) 또는 풀 리퀘스트를 고려해 주세요. 해당 모듈이 기본 제공 세트에 포함될 수 있습니다.

적절한 형식의 예를 보려면 디렉토리 [AoT/aot/inputs](https://github.com/AoT-inc/AoT-AI/tree/master/aot/inputs/)에 있는 기본 제공 모듈을 열어보세요.

또한, 디렉토리 [AoT/aot/inputs/examples](https://github.com/AoT-inc/AoT-AI/tree/master/aot/inputs/examples)에는 사용자 정의 입력 예제가 있습니다.

기본 제공 세트에 포함되지 않은 사용자 정의 모듈에 전념하는 또 다른 GitHub 저장소는 [aot-inc/AoT-custom](https://github.com/AoT-inc/AoT-AI-custom)에서 확인할 수 있습니다.

### 입력 명령

입력 명령은 웹 UI에서 실행할 수 있는 입력 모듈 내의 함수입니다. 이는 보정 또는 입력에 특정한 기타 기능과 같은 작업에 유용합니다. 기본적으로 최소 하나의 작업, 즉 "지금 측정값 획득"이 있으며, 이는 다음 주기가 경과하기 전에 입력이 측정값을 획득하도록 합니다.

!!! 참고
    작업은 입력이 활성 상태일 때만 실행할 수 있습니다.

### 입력 작업

각 주기마다 입력은 측정값을 획득하여 시계열 데이터베이스에 저장합니다. 측정값 획득 후, 하나 이상의 [작업](Actions.md)을 실행하여 입력의 기능을 확장할 수 있습니다. 예를 들어, MQTT 게시 작업을 사용하여 측정값을 MQTT 서버에 게시할 수 있습니다.

### 입력 옵션

입력 옵션에 대한 자세한 내용은 아래 표를 참조하세요.

| 설정                     | 설명                                                                                                                                                                                                 |
|--------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 활성화                   | 센서가 올바르게 구성된 후 활성화하면 센서로부터 측정값을 획득하기 시작합니다. 활성화된 조건부 함수도 작동을 시작합니다.                                                                                   |
| 비활성화                 | 비활성화하면 센서로부터 측정값을 획득하는 작업이 중지됩니다. 관련된 모든 조건부 함수도 작동을 멈춥니다.                                                                                               |
| 저장                     | 특정 센서에 대해 입력 상자에 입력된 현재 구성을 저장합니다.                                                                                                                                           |
| 삭제                     | 특정 센서를 삭제합니다.                                                                                                                                                                           |
| 지금 측정값 획득         | 입력을 강제로 측정하고 데이터베이스에 저장합니다.                                                                                                                                                   |
| 위/아래                  | 특정 센서를 표시된 순서에서 위 또는 아래로 이동합니다.                                                                                                                                               |
| 전원 출력                | 센서를 전원 공급하는 출력을 선택합니다. 이는 센서가 3회 연속 오류를 반환할 경우 문제를 해결하기 위해 전원을 껐다가 다시 켜는 전원 사이클링을 가능하게 합니다. 릴레이 대신 트랜지스터를 사용할 수도 있습니다. |
| 위치                     | 사용 중인 센서에 따라 일련 번호, GPIO 핀, I2C 주소 등을 선택해야 합니다.                                                                                                                             |
| I2C 버스                 | I2C 주소와 통신하는 데 사용할 버스입니다.                                                                                                                                                           |
| 주기(초)                 | 센서가 성공적으로 읽히고 데이터베이스 항목이 생성된 후, 센서를 다시 측정하기까지 대기하는 시간입니다.                                                                                                   |
| 측정 단위                | 측정값을 저장할 단위를 선택합니다(선택 가능한 측정값에만 해당).                                                                                                                                       |
| 사전 출력                | 측정값을 얻기 전에 출력을 활성화해야 하는 경우, 활성화될 출력 번호를 지정합니다.                                                                                                                       |
| 사전 출력 지속 시간(초)  | 센서 측정이 이루어지기 전에 사전 출력이 실행되는 시간입니다.                                                                                                                                           |
| 측정 중 사전 출력        | 활성화된 경우, 측정값을 획득하는 동안 사전 출력이 계속 켜져 있습니다. 비활성화된 경우, 측정값을 획득하기 직전에 사전 출력이 꺼집니다.                                                                       |
| 명령                     | 리눅스 명령(사용자 'root'로 실행됨)의 반환값이 측정값이 됩니다.                                                                                                                                       |
| 명령 측정값              | 리눅스 명령에서 측정된 조건(예: 온도, 습도 등)입니다.                                                                                                                                               |
| 명령 단위                | 리눅스 명령에서 측정된 조건의 단위입니다.                                                                                                                                                           |

위 표는 일부 옵션만 포함하며, 전체 목록은 원문을 참조하세요.
