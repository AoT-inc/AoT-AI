## 오류 코드

AoT는 여러 가지 오류를 반환할 수 있습니다. 아래는 발생할 수 있는 번호가 매겨진 오류와 문제를 진단하는 방법에 대한 정보입니다.

### 오류 100

> '***X***' 값의 유형 ***Y***을(를) 설정할 수 없습니다. 값은 float 또는 float을 나타내는 문자열이어야 합니다.

 - 예시:
  - '***1.33.4***' 값의 유형 ***str***을(를) 설정할 수 없습니다.
  - '***Output: 1.2***' 값의 유형 ***str***을(를) 설정할 수 없습니다.
  - '***[1.3, 2.4]***' 값의 유형 ***list***을(를) 설정할 수 없습니다.
  - '***{"output": 1.99}***' 값의 유형 ***dict***을(를) 설정할 수 없습니다.
  - '***None***' 값의 유형 ***Nonetype***을(를) 설정할 수 없습니다.

이 오류는 InfluxDB 시계열 데이터베이스에 저장하려는 값이 숫자 값(정수 또는 소수/float)이 아니거나 float을 나타내는 문자열(예: "5", "3.14")이 아닌 경우 발생합니다. 이 오류가 발생하는 가장 일반적인 이유는 Input이 센서를 읽을 때 측정값을 반환하지 않거나 숫자 값을 나타내지 않는 데이터를 반환하여 센서가 작동하지 않음을 나타내는 경우입니다. 이는 결함 있는 배선, 불충분한 전원 공급, 결함 있는 센서, I2C 버스가 활성화되지 않음, 잘못된 설정 등 여러 가지 이유로 인해 발생할 수 있습니다. 종종 센서는 데몬이 시작될 때 Input 초기화 중에 실패하거나 올바르게 설정되지 않아 매 측정 주기마다 이 오류가 발생할 수 있습니다. 데몬 로그(`관리 -> 시스템 로그`)를 데몬이 시작된 시점까지 검토하여 초기화 오류를 확인해야 합니다. 컨트롤러 설정에서 로그 레벨: 디버그를 활성화하면 디버깅 로그 라인(사용 가능한 경우)을 추가로 제공하여 문제를 해결하는 데 유용할 수 있습니다.

### 오류 101

> ***X***이(가) 제대로 설정되지 않았습니다.

 - 예시:
  - 장치가 설정되지 않음
  - 출력 채널 ***Y***이(가) 설정되지 않음

이 오류는 컨트롤러(Input/Output/Function 등)가 시작될 때 장치나 채널을 제대로 초기화하지 못했으며, 이제 초기화되지 않은 장치나 채널에 접근하려고 할 때 발생합니다. Input의 경우, 센서와 통신하는 데 사용되는 3rd-party 라이브러리를 로드하는 과정에서 문제가 발생할 수 있습니다. 라이브러리를 로드하는 데 오류가 있으면 센서와 통신할 수 없습니다. 컨트롤러가 처음 활성화되었을 때 발생한 관련 오류를 확인하려면 데몬 로그(`관리 -> 시스템 로그`)를 검토해야 합니다. 장치를 비활성화한 후 다시 활성화하여 초기화 오류를 다시 확인해 보십시오. 컨트롤러 설정에서 로그 레벨: 디버그를 활성화하면 디버깅 로그 라인(사용 가능한 경우)을 추가로 제공하여 문제를 해결하는 데 유용할 수 있습니다.
