페이지: `설정 -> 출력`

지원되는 출력 장치의 전체 목록은 [지원되는 출력 장치](Supported-Outputs.md)를 참조하세요.

출력은 장치를 작동시키는 다양한 신호를 생성할 수 있습니다. 출력은 GPIO 핀에서 HIGH/LOW 신호, 펄스 폭 변조(PWM) 신호, 315/433 MHz 신호를 사용하여 무선 주파수로 작동하는 릴레이를 전환하거나, 펌프 및 모터를 구동하거나, Linux 또는 Python 명령을 실행하는 등의 작업을 수행할 수 있습니다.

## 사용자 정의 출력

AoT에는 사용자 정의 출력을 AoT 시스템에서 생성하고 사용할 수 있는 사용자 정의 출력 가져오기 시스템이 있습니다. 사용자 정의 출력은 `[기어 아이콘] -> 구성 -> 사용자 정의 출력` 페이지에서 업로드 및 가져오기 할 수 있습니다. 가져온 후에는 `설정 -> 출력` 페이지에서 사용할 수 있습니다.

작동하는 모듈을 개발한 경우 [새 GitHub 이슈 생성](https://github.com/aot-inc/AoT/issues/new?assignees=&labels=&template=feature-request.md&title=New%20Module) 또는 풀 리퀘스트를 고려해 보세요. 해당 모듈이 기본 제공 세트에 포함될 수 있습니다.

적절한 형식의 예는 [AoT/aot/outputs](https://github.com/aot-inc/AoT/tree/master/aot/outputs/) 디렉토리에 있는 기본 제공 모듈을 열어 확인할 수 있습니다. 또한 [AoT/aot/outputs/examples](https://github.com/aot-inc/AoT/tree/master/aot/outputs/examples) 디렉토리에는 사용자 정의 출력 예제가 포함되어 있습니다.

새로운 측정값/단위가 필요한 출력의 경우 `[기어 아이콘] -> 구성 -> 측정값` 페이지에서 추가할 수 있습니다.

## 켜기/끄기 (GPIO)

켜기/끄기 (GPIO) 출력은 GPIO 핀을 HIGH (3.3볼트) 또는 LOW (0볼트)로 전환합니다. 이는 릴레이와 같은 전기 기계식 스위치를 제어하여 전기 장치를 켜고 끄는 데 유용합니다.

릴레이는 소형 전압 신호(예: 마이크로프로세서에서 발생)를 사용하여 훨씬 더 큰 전압을 활성화할 수 있는 전기 기계식 또는 솔리드 스테이트 장치로, 저전압 시스템이 고전압의 위험에 노출되지 않도록 합니다.

출력 탭에서 출력을 추가하고 구성하세요. 출력은 시스템의 다른 부분에서 사용되기 전에 올바르게 설정되어야 합니다.

유선 릴레이를 설정하려면 "GPIO 핀"(BCM 번호 체계 사용)을 HIGH (5볼트) 및 LOW (0볼트)로 전환할 핀으로 설정하세요. 이는 릴레이 및 기타 장치를 활성화하는 데 사용됩니다. *켜기 트리거*는 장치를 켜는 신호 상태(HIGH 또는 LOW)로 설정해야 합니다. 예를 들어, 릴레이가 코일의 전위가 0볼트일 때 활성화된다면 *켜기 트리거*를 "LOW"로 설정하세요. 반대로 릴레이가 코일의 전위가 5볼트일 때 활성화된다면 "HIGH"로 설정하세요.

## 펄스 폭 변조 (PWM)

펄스 폭 변조(PWM)는 특정 주파수(Hz)에서 펄싱 신호에 메시지를 인코딩하는 변조 기술입니다. 부하에 공급되는 전압(및 전류)의 평균 값은 전원과 부하 사이의 스위치를 빠른 속도로 켜고 끔으로써 제어됩니다. 스위치가 꺼져 있는 시간에 비해 켜져 있는 시간이 길수록 부하에 공급되는 총 전력이 높아집니다.

PWM 스위칭 주파수는 부하(전력을 사용하는 장치)에 영향을 미칠 수 있는 주파수보다 훨씬 높아야 합니다. 즉, 부하가 인식하는 결과 파형은 가능한 한 매끄러워야 합니다. 전원 공급 스위칭 속도는 부하와 응용 프로그램에 따라 크게 달라질 수 있습니다. 예를 들어:

!!! 인용
    전기 스토브에서는 분당 여러 번 스위칭해야 하고, 램프 디머에서는 120 Hz, 모터 드라이브에서는 몇 kHz에서 수십 kHz, 오디오 앰프 및 컴퓨터 전원 공급 장치에서는 수십 또는 수백 kHz에 이릅니다.

듀티 사이클은 '켜짐' 시간과 정기적인 간격 또는 '주기' 시간의 비율을 설명합니다. 낮은 듀티 사이클은 대부분의 시간 동안 전원이 꺼져 있기 때문에 낮은 전력을 의미합니다. 듀티 사이클은 0%는 항상 꺼짐, 50%는 절반의 시간 동안 꺼짐 및 절반의 시간 동안 켜짐, 100%는 항상 켜짐으로 표현됩니다.

## 연동 펌프(Peristaltic Pump)

AoT는 두 가지 연동 펌프 출력 모듈을 지원합니다. 일반 연동 펌프 출력(Generic Peristaltic Pump Output)과 Atlas Scientific EZO-PMP 연동 펌프입니다.

### 일반 연동 펌프

일반 연동 펌프 출력을 사용하여 모든 연동 펌프를 이용해 액체를 분배할 수 있습니다. 가장 기본적인 분배 기능은 분배 시작, 분배 중지, 또는 일정 시간 동안 분배하는 것입니다. 펌프 속도가 측정된 경우, 이 값을 "최고 속도(ml/min)" 설정에 입력하면 출력 컨트롤러가 단순히 시간 대신 특정 부피를 분배할 수 있습니다. 특정 부피를 분배하려면 출력 모드를 설정하고 "원하는 유량(ml/min)"을 지정해야 합니다.

펌프의 유량을 측정하려면 먼저 펌프 호스에서 모든 공기를 제거하세요. 그런 다음 펌프가 60초 동안 분배하도록 지시하고 분배된 액체를 수집합니다. 완료되면 분배된 액체의 양을 밀리리터 단위로 측정하여 "최고 속도(ml/min)" 설정에 입력하세요. 펌프의 유량이 설정되면 이제 특정 부피를 분배할 수 있습니다.

이 출력 모듈은 GPIO 핀을 HIGH 또는 LOW로 전환하여 연동 펌프를 켜고 끄는 방식으로 작동합니다. 이는 펌프의 전원 공급 장치와 직렬로 릴레이를 사용하거나 GPIO를 펌프의 입력 신호로 직접 사용하는 방식으로 가장 쉽게 구현할 수 있습니다(지원되는 경우). 릴레이를 사용할 때는 펌프의 빠른 전환을 가능하게 하는 회로를 설계하는 것이 중요합니다. 펌프가 분배하는 부피는 시간에 따라 달라지므로, 펌프 전환 속도가 빠를수록 분배 정확도가 높아집니다. 많은 연동 펌프는 DC 전압으로 작동하며 AC-DC 변환기가 필요합니다. 이러한 변환기는 전원이 공급되거나 제거될 때 에너지가 활성화되거나 비활성화되는 데 시간이 걸릴 수 있으며, 이는 분배 정확도에 영향을 미칠 수 있습니다. 이를 해결하려면 AC 전원이 아닌 DC 전원을 전환하여 이러한 잠재적 지연을 제거해야 합니다.

### Atlas Scientific 연동 펌프

Atlas Scientific 연동 펌프는 I2C 또는 직렬 통신을 통해 특정 부피의 액체를 정확히 분배할 수 있는 연동 펌프와 마이크로컨트롤러가 결합된 장치입니다. 이 펌프는 보정, 켜기, 끄기, 특정 속도로 분배 등의 명령을 포함한 [여러 명령](https://www.atlas-scientific.com/files/EZO_PMP_Datasheet.pdf)을 수락할 수 있습니다. Atlas Scientific 연동 펌프는 좋은 옵션이지만, 일반 연동 펌프보다 비용이 더 많이 듭니다.

## 무선 315/433 MHz

일부 315/433 MHz 무선 릴레이를 사용할 수 있습니다. 송신기의 핀(BCM 번호 체계 사용), 펄스 길이, 비트 길이, 프로토콜, 켜기 명령, 끄기 명령을 설정해야 합니다. 켜기 및 끄기 명령을 확인하려면 315/433 MHz 수신기를 Pi에 연결한 후 아래 스크립트를 실행하세요. 이때 17을 수신기가 연결된 핀(BCM 번호 체계 사용)으로 바꾸고, 리모컨의 버튼(켜기 또는 끄기)을 눌러 해당 버튼과 연관된 숫자 코드를 감지합니다.

```bash
sudo /opt/AoT/env/bin/python /opt/AoT/aot/devices/wireless_rpi_rf.py -d 2 -g 17
```

433 MHz 무선 릴레이는 SMAKN 433MHz RF 송신기/수신기 및 Etekcity 무선 원격 제어 전기 콘센트와 성공적으로 테스트되었습니다([이슈 88](https://github.com/aot-inc/AoT/issues/88) 참조). 현재 코드로 작동하지 않는 315/433 MHz 송신기/수신기 및 무선 릴레이가 있는 경우, 하드웨어 세부 정보를 포함하여 [새 이슈](https://github.com/aot-inc/AoT/issues/new)를 제출하세요.

## Linux 명령

출력을 켜거나 끄거나 듀티 사이클을 설정할 때 터미널 명령을 실행하는 옵션입니다. 명령은 'root' 사용자로 실행됩니다. Linux 명령 출력을 생성하면 출력 사용 방법을 보여주는 예제 코드가 제공됩니다.

## Python 명령

Python 명령 출력은 Linux 명령 출력과 유사하게 작동하지만 Python 3 코드를 실행합니다. Python 명령 출력을 생성하면 출력 사용 방법을 보여주는 예제 코드가 제공됩니다.

## 출력 참고 사항

무선 및 명령(Linux/Python) 출력: 무선 프로토콜은 315/433 MHz 장치와의 단방향 통신만 허용하므로, 무선 릴레이는 켜지기 전까지는 꺼진 상태로 간주되며, 추가되었을 때 빨간색(꺼짐)으로 표시됩니다. 무선 릴레이가 AoT 외부(예: 리모컨)에서 켜지거나 꺼진 경우, AoT는 릴레이의 상태를 확인할 수 없으며 마지막 상태를 표시합니다. 예를 들어, AoT가 무선 릴레이를 켜고 리모컨으로 릴레이를 끄면, AoT는 여전히 릴레이가 켜져 있다고 가정합니다.
